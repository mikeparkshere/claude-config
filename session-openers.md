# Session Openers

The opener primes Claude with what already exists. The closer adds to what exists. Together they're the loop that compounds.

---

## Session #1 — Brand New Project

For the first session on a brand new project, use this opener:

> Read CLAUDE.md and all the files in /docs/ before we start. This is our first session on this project — no memory exists yet, so prime fully from those files.

It's a small distinction but it sets the right expectation: Claude won't be confused looking for memory that doesn't exist, and it knows it's responsible for building memory during this session that future sessions will inherit.

---

## Session #2 Onward — Standard Opener

After session #1, switch to the standard opener:

> Check memory first. Read CLAUDE.md and the files in /docs/ before we start.

---

## The Closer (Just as Important)

Don't forget the bookend. As you wrap each session:

> Before we close, what's worth preserving from this session that isn't already in CLAUDE.md or /docs/? Distill anything new into the right file.

That's the line that makes the next session start hot.
